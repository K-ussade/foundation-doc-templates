# Versioning Policy

## Purpose

This policy defines how template changes are versioned and communicated.

## Version Classes

| Class | Meaning |
| --- | --- |
| Patch | Wording, formatting, link, or validation corrections that do not change required sections. |
| Minor | Adds optional sections, examples, or guidance without changing required gates. |
| Major | Changes required sections, authority language, boundary rules, release status rules, or validation expectations. |

## Change Requirements

| Change | Requirement |
| --- | --- |
| Patch | Update affected template and run validation. |
| Minor | Update template guide and validation if needed. |
| Major | Update template guide, boundary rules, quality gate, graph map, and downstream adoption notes. |

## Public Compatibility

Existing repositories do not automatically inherit template changes. Each repo must adopt updates through its own review and validation process.
