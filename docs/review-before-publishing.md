# Review Before Publishing

## Purpose

This checklist defines the review gates required before publishing a document created from these templates.

## Required Gates

| Gate | Reviewer | Required outcome |
| --- | --- | --- |
| Entity voice review | Human owner | Foundation, company, product, and personal-account language remain separate. |
| Status review | Human owner | Planned, scaffolded, released, draft, or private/not-public status is explicit. |
| Privacy review | Human owner | Donor, student, volunteer, customer, private operations, and private training material are absent. |
| Security review | Human owner | Secrets, tokens, exact sensitive infrastructure locations, and NEURONA operational details are absent. |
| Release review | Release authority | No model, dataset, Space, school, NEURONA, deployment, or service is claimed without public artifacts. |
| Portfolio review | Alexandra | Any future Upwork or portfolio claim is evidence-backed and approved. |

## Publication Rule

Passing the template validator is required but not sufficient for publication. Human review remains the authority for public, Foundation-facing, client-facing, and portfolio-facing material.

## Required Exclusion Terms

Publication review must explicitly exclude donor private data, student data, volunteer private data, customer data, private Foundation operations, private training material, sealed YOSO-YAi LLC IP, secrets, tokens, exact sensitive infrastructure locations, and security-sensitive NEURONA operational details.
