# Reuse Policy

## Purpose

This policy defines how other public Foundation repositories reuse templates without importing private material or unreviewed claims.

## Reuse Rules

| Rule | Standard |
| --- | --- |
| Copy public-safe structure only | Do not copy private drafts or repo-specific sensitive content. |
| Replace placeholders | Do not leave ambiguous placeholders in public-facing repo roots. |
| Add local boundary rules | Every repo must list its own forbidden material. |
| Validate locally | Every repo needs its own validation script or documented check. |
| Preserve status | Planned, scaffolded, experimental, released, and deprecated terms must stay accurate. |
| Update graphs | Local graphs must match actual authority, workflow, and boundary behavior. |

## Refusal Rules

Refuse reuse when a template would:

- imply a release that has not occurred;
- collapse Foundation and company voices;
- expose private or sealed material;
- turn a public scaffold into a raw data or model artifact store;
- skip human review for governance, safety, data, or release claims.
