# Template Guide

## Purpose

This guide explains how Foundation repositories should use these templates.

## Use Rules

| Rule | Standard |
| --- | --- |
| Start with owner and authority | Every public repo must identify Foundation owner and human authority. |
| Mark status | Use planned, scaffolded, experimental, released, or deprecated. |
| Preserve boundaries | Keep public, private, sealed, and release-only material explicit. |
| Use placeholders | Templates must use `TODO` or clearly generic examples. |
| Avoid real private examples | Never include real donor, student, school, volunteer, customer, incident, telemetry, or sealed material. |
| Update graphs | Add Mermaid maps when authority, workflow, release, or boundary behavior changes. |

## Template Adoption Steps

1. Copy the relevant template into the target repository.
2. Replace `TODO` placeholders with public-safe content.
3. Add repository-specific forbidden material.
4. Add or update validation.
5. Run validation before publication.

## Review Requirement

Human Foundation authority must review template-derived public documents before they are treated as approved policy, report, release, or safety material.
