# Public / Private Boundaries

## Purpose

This document defines the boundary rules for every template and example in this repository.

## Public-Safe Material

Allowed public material includes:

- template structure;
- generic governance and reporting language;
- synthetic examples clearly labeled synthetic/public-safe;
- public repository links;
- public status vocabulary;
- public-safe Mermaid diagrams;
- release-readiness checklists that do not claim releases exist.

## Private Or Blocked Material

Do not include:

- donor private data;
- student data;
- school private data;
- volunteer private data;
- private Foundation operations;
- private partner records;
- private legal drafts or privileged material;
- customer data;
- private training corpora;
- raw datasets;
- model weights;
- private evaluations;
- sealed YOSO-YAi LLC IP;
- secrets, tokens, API keys, or credentials;
- exact sensitive infrastructure locations;
- security-sensitive NEURONA operational details.

## Mixed-Source Handling

If a document is adapted from a mixed or private source, the public template output must be rewritten from approved public facts. Summaries, screenshots, examples, generated outputs, and logs inherit the source boundary until human review clears them for public use.
